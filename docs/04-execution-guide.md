# 执行步骤指南

## 1. 开发工作流

### 1.1 标准流程
```
1. 需求分析 → 明确用户需求与约束
2. 设计 → 确定视觉方案和组件结构
3. 实现 → 逐组件编码
4. 审查 → Review 代码质量和视觉一致性
5. 部署 → 静态部署
```

### 1.2 提交规范
```
feat: 新功能
fix: 修复
docs: 文档
style: 样式
refactor: 重构
chore: 杂项
```

## 2. 开发日志使用

### 2.1 创建日志
每次工作开始/结束时更新 `dev-log/YYYY-MM-DD.md`

### 2.2 日志模板
```markdown
# 开发日志 YYYY-MM-DD

## 今日完成
- [ ] 任务 1
- [ ] 任务 2

## 待办事项
- [ ] 明日任务 1
- [ ] 明日任务 2

## 技术笔记
- 遇到的问题和解决方案
```

### 2.3 日志原则
- 每天一个文件
- 完成项用 `[x]` 标记
- 技术决策记录原因

## 3. AI 协作规范

### 3.1 CLAUDE.md 指引
项目根目录维护 `CLAUDE.md`，内容包含：
- 项目标准文件路径映射
- 工作模式说明
- 使用的设计语言引用

### 3.2 AI 工作模式
| 模式 | 触发词 | 行为 |
|------|--------|------|
| 设计审查 | /review | 审查当前变化 |
| 技术问答 | /grill | 追问技术决策 |
| 架构改进 | /architecture | 提出改进建议 |

## 4. 质量标准

### 4.1 代码审查清单
- [ ] 视觉匹配 Apple 设计系统
- [ ] 响应式测试通过（320/734/1068/1440px）
- [ ] 动画流畅（60fps）
- [ ] 无控制台错误
- [ ] HTML 语义化验证通过
- [ ] CSS 变量命名一致

### 4.2 性能基准
- HTML 大小 < 20KB
- CSS 大小 < 30KB
- JS 大小 < 15KB
- 总请求数 < 5
