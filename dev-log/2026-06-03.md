## 技术笔记

### 设计决策记录
- **决策**：使用纯 CSS Custom Properties 实现 Apple 设计系统
- **原因**：零依赖、运行时可替换、语义化命名
- **替代方案**：Sass Variables（需要构建步骤）、Tailwind（过于抽象）

- **决策**：Vanilla HTML/CSS/JS，不使用框架
- **原因**：单页应用无需框架开销，零外部依赖保证加载速度
- **替代方案**：React/Vue（过度工程化）

### 架构要点
1. CSS 采用设计 tokens 体系，所有视觉常量集中管理在 `:root` 中
2. 使用 IntersectionObserver 实现高性能滚动动画
3. 响应式断点按照 Apple 标准：734px / 1068px
4. 深色模式通过 `prefers-color-scheme` 自动适配

### Apple 设计语言关键点
- 唯一强调色 `#0066cc`（Action Blue）
- 负字距排版 `-0.374px` ~ `-0.28px`
- 无装饰性边框/阴影，用底色变化做分隔
- SF Pro Display / SF Pro Text 字体体系
- 导航栏 44px 高度

### 遇到的问题
1. PDF 文件解析困难
   - 原因：PDF 使用 FlateDecode 压缩 + CID 字体编码（CJK）
   - 当前状态：已提取 CMap 映射，可部分解析文本内容
   - 待优化：后续可集成 pdf.js 实现完整解析
